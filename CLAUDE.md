# Watch Microbrand Database — Claude Instructions

## Project Overview

A monitoring database of microbrand and startup watch brands from across the world. Companion to the `watch-brand-location-database` project (which maps luxury/independent ateliers for tour visits).

**Primary use:** Outreach for The Watch Collectors' Club watch shows (UK).

## Architecture

- **Source of truth:** JSON files in `data/` — always edit these, never edit the spreadsheet directly
- **Output:** `watch-microbrand-database.xlsx` — regenerated by `node scripts/build-spreadsheet.js`
- **Pattern:** Same as watch-brand-location-database project

## Data Files

| File | Region |
|------|--------|
| `data/microbrands-europe.json` | UK, France, Germany, Italy, Scandinavia, Benelux, etc. |
| `data/microbrands-americas.json` | USA, Canada, South America |
| `data/microbrands-asia-pacific.json` | Japan, China, Singapore, Australia, Hong Kong, etc. |
| `data/microbrands-other.json` | Middle East, Africa, unassigned |

## Schema

Each entry has these fields:
```json
{
  "brandName": "string",
  "country": "string",
  "townCity": "string or null",
  "foundedYear": "number or null",
  "priceRangeLow": "number or null (USD)",
  "priceRangeHigh": "number or null (USD)",
  "status": "Active | Dormant | Defunct",
  "latestModel": "string or null",
  "lastActivityDate": "string (ISO date) or null",
  "website": "string or null",
  "instagramHandle": "string or null (without @)",
  "source": "MBWDB | existing-db | manual",
  "notes": "string or null"
}
```

## Status Definitions

- `Active` — confirmed selling watches in the last 12 months
- `Dormant` — no confirmed activity for 12+ months but not confirmed closed
- `Defunct` — website down, socials dead, no activity for 2+ years

## Constraints

- Price ceiling: under $5,000 USD per watch
- Exclude Kickstarter-only brands (delivery must have completed before inclusion)
- Only include brands where watches are actively for sale or have been sold
- No group-owned / conglomerate brands

## Workflow

1. Edit JSON files in `data/`
2. Sort entries alphabetically by `brandName` before saving
3. Run `node scripts/build-spreadsheet.js` to regenerate the Excel file
4. Commit both the JSON changes and the rebuilt spreadsheet

## Scripts

- `build-spreadsheet.js` — generates `watch-microbrand-database.xlsx`
- `seed-from-existing-db.js` — one-shot: extracts microbrand candidates from the 438-brand location DB
- `check-websites.js` — health-checks all website URLs
